### 2.2 时间范围

2. 最近1个月
3. 最近3个月

### 2.3 租户
支持按租户名称或租户 ID 搜索并切换。

当前代码内置 4 个租户示例：
1. 上港长江（tenant-1001）
2. 宁波远洋（tenant-1002）
3. 青岛海联（tenant-1003）
4. 厦门海丰（tenant-1004）

规则：
1. 切换租户后，组织层级树随租户变化。
2. 不同租户下的部门与人员不同，不做同质化处理。
3. 租户筛选作用于全部 KPI、图表、明细和导出。

### 2.4 组织层级
组织层级基于当前租户的组织树选择，默认值为“全公司”。

当前实现规则：

5. 组织层级影响数据范围聚合。

说明：

### 2.5 来源

规则：
1. 使用滑轨式切换样式。

3. “全部来源”场景下，转化分析按邮件/文件拆色展示。

### 2.6 筛选记忆

## 3. KPI 卡片

### 3.1 当前 KPI 数量
当前顶部保留 3 张 KPI 卡片：
1. 工作单提交量
2. 字段一次通过率
3. 节省总成本

### 3.2 KPI 切换逻辑
点击 KPI 卡片切换中部图表和底部明细表模块：
1. 点击“工作单提交量”进入转化分析
2. 点击“字段一次通过率”进入数据质量分析
3. 点击“节省总成本”进入效率成本分析

### 3.3 KPI 展示规则
每张 KPI 卡片展示以下信息：
1. 指标名称
2. 当前值
3. 较上周期变化方向与比例
4. “较上周期”辅助描述

### 3.4 KPI 计算口径
#### 3.4.1 工作单提交量
指标字段：`work_order_submit_volume`  
定义：统计周期内提交成功的工作单总量

#### 3.4.2 字段一次通过率
指标字段：`field_first_pass_rate`  
定义：字段初始值与最终提交值一致的占比，空→空也计入。  
公式：`field_nochange_cnt / field_total_cnt`

#### 3.4.3 节省总成本
定义：基于“原时长”和“当前平均时长”的差值，估算整体节省的人力成本。  
公式：

```text
节省总成本 = 客服节省成本 + 操作节省成本

客服节省成本 = (原校对时长 - 平均每工作单校对总时长) × 工作单提交量 × 客服成本 ÷ 480
操作节省成本 = (原审核时长 - 平均每工作单审核总时长) × 工作单提交量 × 操作成本 ÷ 480
```

说明：
1. `480 = 8h × 60min`，即按 `1人天 = 8小时` 换算。
2. 当前实现允许节省成本为负值，即当现状效率低于原时长基准时，表现为“负节省”。

## 5. 数据质量分析

### 5.3 核心质量指标总览
图表类型：六维雷达图

#### 5.3.1 展示维度
当前展示以下 6 个维度：
1. 文件识别准确率
2. 邮件识别准确率
3. 字段一次通过率
4. 字段未修改率
5. 字段无需补录率
6. 字段保留率

#### 5.3.2 指标定义
1. 文件识别准确率  
   公式：`file_correct_cnt / file_total_cnt`
2. 邮件识别准确率  
   公式：`mail_correct_cnt / mail_total_cnt`
3. 字段一次通过率  
   公式：`field_nochange_cnt / field_total_cnt`
4. 字段未修改率  
   公式：`1 - field_change_rate`
5. 字段无需补录率  
   公式：`1 - field_supplement_rate`
6. 字段保留率  
   公式：`1 - field_missrecall_rate`

#### 5.3.3 字段状态归属规则
字段一次通过率、字段修改率、字段补录率、字段误召回率满足以下互斥划分：

1. 空 → 空：字段一次通过率
2. 有值A → 有值A：字段一次通过率
3. 有值A → 有值B，且 A≠B：字段修改率
4. 空 → 非空：字段补录率
5. 非空 → 空：字段误召回率

因此：

```text
字段一次通过率 + 字段修改率 + 字段补录率 + 字段误召回率 = 1
```

## 9. 关键数据口径汇总

### 9.1 转化类
1. 来源转工单转化率 = `成功创建工单量 / 来源输入量`
2. 漏单率 = `(来源输入量 - 成功创建工单量) / 来源输入量`

### 9.2 质量类
1. 文件识别准确率 = `file_correct_cnt / file_total_cnt`
2. 邮件识别准确率 = `mail_correct_cnt / mail_total_cnt`
3. 字段一次通过率 = `field_nochange_cnt / field_total_cnt`
4. 字段未修改率 = `1 - field_change_rate`
5. 字段无需补录率 = `1 - field_supplement_rate`
6. 字段保留率 = `1 - field_missrecall_rate`

### 9.3 效率类
1. 平均每工作单校对总时长 = `avg_proofreading_duration_per_work_order`
2. 平均每工作单审核总时长 = `avg_audit_duration_per_work_order`
3. 平均每工作单处理总时长 = `avg_processing_duration_per_work_order`
4. 返工率 = `rework_rate`
5. 平均每工作单提交次数 = `avg_submit_times_per_work_order`

### 9.4 成本类
1. 平均每工作单客服成本 = `total_follower_cost / work_order_submit_volume`
2. 平均每工作单操作成本 = `total_reviewer_cost / work_order_submit_volume`
3. 平均每工作单成本 = `total_labor_cost / work_order_submit_volume`
4. 客服总成本 = `客服投入人天 × 客服成本单价`
5. 操作总成本 = `操作投入人天 × 操作成本单价`
6. 总人力成本 = `客服总成本 + 操作总成本`

### 9.5 投入时间类
1. 客服投入时间展示单位为 `h`
2. 操作投入时间展示单位为 `h`
3. 灰字换算规则：`天 = 小时 / 8`

## 10. 当前实现说明

### 10.2 当前实现特征

### 10.3 后续建议
后续若接入真实数据源，建议优先明确以下内容：
1. 时间粒度映射规则（日/周/月）
2. 组织层级是否升级为多选聚合
3. 转化链路每一阶段的真实业务事件定义
4. 字段质量指标所依赖的底层明细表结构
5. 节省成本口径是否允许出现负值，以及是否需要展示为 0 下限
